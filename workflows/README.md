# Workflows — цепочки субагентов

Каждый файл описывает последовательность запусков субагентов для решения конкретной задачи.

Правила:
- Выполняй шаги строго по порядку.
- Следующий шаг — только после сохранения результата предыдущего в `output/`.
- Если шаг упирается в «данных недостаточно» — фиксируй и переходи к следующему.

| Файл | Когда использовать |
|---|---|
| `workflow_segment_deep_dive.md` | Глубокий анализ одного сегмента (воронка → интервью → гипотезы) |
| `workflow_hypothesis_sprint.md` | Быстрая генерация и приоритизация гипотез по всем сегментам |
| `workflow_pitch_prep.md` | Подготовка питча: от данных до слайдов |
