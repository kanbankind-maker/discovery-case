# Workflow: Pitch Prep

**Цель:** Подготовить питч для одного сегмента на основе всех данных  
**Время:** ~20–30 мин (если шаги 1–3 Segment Deep Dive уже выполнены)  
**Требует:** Наличия output-файлов по сегменту (funnel + insights + hypotheses)

---

## Предусловия

Проверь, что в `output/` есть:
- `*funnel-[segment]*`
- `*insights-[segment]*` (или пропусти если нет интервью)
- `*hypotheses-[segment]*`

Если файлов нет → сначала пройди `workflow_segment_deep_dive.md`.

---

## Шаги

### Шаг 1 — Напиши питч
- Используй: `.cursor/agents/agent_pitch_writer.md`
- ✅ Результат в `output/*pitch-[segment]*`

### Шаг 2 — Самопроверка (ручной шаг)
Проверь питч по чеклисту:
- [ ] Есть ли Problem Statement с данными (не «пользователям неудобно»)?
- [ ] Есть ли хотя бы 1 фальсифицируемая гипотеза?
- [ ] Есть ли раздел «Риски» с митигацией?
- [ ] Все цифры имеют источник?
- [ ] Объём ≤ 400 слов?

### Шаг 3 — Challenge-вопрос
Задай агенту:
> «Найди слабейший аргумент в этом питче и предложи как его усилить»

Зафиксируй ответ в том же файле (раздел `## Weak Points`).

---

## Чеклист завершения

- [ ] Питч создан в `output/`
- [ ] Все 5 пунктов самопроверки выполнены
- [ ] Weak points зафиксированы
