# Стратегия T-Pay Marketplace

## Миссия

Сделать финансовую экосистему Т-Банка точкой входа в e-commerce для 15M активных пользователей — за счёт доверия к банку, персонализации через финансовые данные и встроенного платёжного опыта без трения.

---

## OKR Q1 2026

### Objective: Вырасти x3 по конверсии первой покупки

| Key Result | Целевое значение | Базовое значение |
|---|---|---|
| KR1: Конверсия первой покупки | 6.3% | 2.1% |
| KR2: Отвал на шаге Товар → Корзина | ≤50% | 77% |
| KR3: Доля первых покупок без промокода | ≥40% | 24–35% |
| KR4: Конверсия упрощённого чекаута (<15k руб.) | +15 п.п. к базе | — |

> **Примечание по KR3.** Сейчас 65–76% первых покупок совершаются с промокодом. Высокая промозависимость снижает unit-экономику и не формирует органическую привычку. Рост доли покупок без промо — индикатор здоровья воронки.

## Стратегические ставки

### 1. Marketplace Discovery UX
Пользователь попадает в T-Pay через баннер внутри банковского приложения. Текущий UX discovery не адаптирован под сегменты: нет персонализированных подборок, нет контекстных рекомендаций на основе финансового профиля. Ставка: внедрить алгоритмическую витрину с персонализацией для сегментов B (Семья) и E (Инвесторы) — без дополнительного согласия (разрешено политикой персональных данных банка).

### 2. Trust-Building для новых пользователей
Основной барьер первой покупки — недоверие к маркетплейсу внутри банковского приложения: пользователи опасаются за безопасность транзакции и возврат средств. Ставка: социальные доказательства (рейтинги, отзывы), видимые гарантии возврата и банковская страховка покупки на карточке товара.

### 3. Promo Efficiency
Промокоды эффективно конвертируют, но формируют зависимость. Ставка: A/B-тестирование механик без промо (кешбэк баллами Т-Банка, рассрочка 0%, персональные предложения по финансовому профилю) с целью снизить долю промо-конверсий до 60% к концу Q1.

### 4. Сегментные флоу
Единый UX не работает для аудитории 18–65+. Ставка: разработать 2–3 дифференцированных сценария онбординга и карточки товара по ключевым сегментам:
- **A (Молодёжь):** акцент на тренды, быстрый чекаут, BNPL
- **B (Семья):** подборки для дома, рассрочка, гарантии возврата
- **C (Пенсионеры):** упрощённый UI, телефонная поддержка, крупный шрифт
- **D (МСБ/ИП):** B2B-товары, ЭДО, закупочные сценарии
- **E (Инвесторы):** персонализация по доходности, эксклюзивные оферы

---

## Ключевые ограничения

- Упрощённый чекаут юридически разрешён только для транзакций < 15 000 руб. (82% всех транзакций — в этом диапазоне).
- Декабрьский всплеск (новогодние подарки) — краткосрочный эффект; стратегия не должна зависеть от сезонности.
- Персонализация через банковские данные без дополнительного согласия допустима только для сегментов B и E.
