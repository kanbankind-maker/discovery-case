# Discovery Case: T-Pay Marketplace

T-Pay — маркетплейс, встроенный в банковское приложение Т-Банк (15M MAU). Цель воркшопа: найти рычаги роста конверсии первой покупки с 2.1% до 6.3% в Q1 2026. Главный провал воронки — переход Товар → Корзина (77% отвала). Работаем с 5 сегментами (A–E) через данные, гипотезы и агентные инструменты.

## Структура репозитория

| Папка / файл | Содержимое |
|---|---|
| `.cursor/rules/role.mdc` | Роль PM, контекст продукта, правила агента |
| `company/strategy.md` | Миссия, OKR, стратегические ставки |
| `company/market_context.md` | Рынок, конкуренты, регуляторика |
| `research/segments.md` | Профили 5 сегментов (A–E) |
| `research/qualitative.md` | Инсайты интервью, Jobs To Be Done |
| `research/quantitative.md` | Данные воронки, метрики, разбивка по сегментам |
| `hypotheses/backlog.md` | Бэклог гипотез для тестирования |
| `output/` | Результаты работы агентов (сохранять сюда) |
